How should the condition bars encode data?

Each row in the departure table gets a mini-timeline bar. What should it show?

A

SOG Bar (height) + Wind Color

Bar height = speed over ground. Color = wind strength (blue→green→yellow→red). Gives two dimensions in one bar. Motoring periods shown as striped/hatched.

Fri 06:00 → Sun 14:00 (56h)   SOG: height   TWS: color
🟢 8-15kt 🟡 15-20kt 🔴 20kt+ ▨ Motor
B

Simple Color Blocks (Composite Score)

Combine SOG, TWS, swell, and motoring into a single "sailing quality" score per time segment. Green = great, yellow = okay, red = rough. Simpler to read but hides detail.

Fri 06:00 → Sun 14:00 (56h)   Block = 3h segment quality
🟢 Good 🟡 Marginal 🔴 Rough 🔵 Motor
C

Switchable Bar Mode

Small toggle above the table: SOG | TWS | TWA | Swell. The bars change to show whichever metric the user cares about. Color thresholds adjust per metric. Power-user friendly.

SOG TWS TWA Swell
Showing: Speed Over Ground (kt)
